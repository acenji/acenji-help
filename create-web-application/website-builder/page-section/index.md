# Page Section

The **Page Section** in the Website Builder is your central hub for managing all pages in your web application. From here, you can create, organize, and configure the pages that make up your application's structure.

---

## Overview

The Page Section enables you to:

- **Create new pages** for your application
- **Manage existing pages** (rename, duplicate, delete)
- **Organize page structure** and navigation
- **Configure page settings** and layouts
- **Navigate between pages** while building

---

## Accessing the Page Section

The Page Section is located in the left panel of the Website Builder:

1. Open your project in the **Website Builder**
2. Look for the **Pages** panel on the left side
3. Click to expand or collapse the page list

## Page Properties

Each page has configurable properties:

| Property | Description |
|----------|-------------|
| **Name** | Display name for the page |
| **URL Slug** | URL path segment (e.g., `/about`, `/dashboard`) |
| **Title** | Browser tab title |
| **Description** | SEO description for the page |
| **Layout** | Page layout configuration |
| **Access** | Visibility and permissions settings |

---

## Working with Pages

### Creating a New Page

1. Click the **"+ Add Page"** button
2. Enter a page name
3. Configure initial settings
4. Click **Create**
5. The new page is added and selected

### Selecting a Page

1. Click on a page name in the list
2. The page loads in the canvas
3. Page becomes the active editing context
4. Floating Design Panel shows page options

### Page Actions

Right-click or use the menu on a page to access:

| Action | Description |
|--------|-------------|
| **Rename** | Change the page name |
| **Duplicate** | Create a copy of the page |
| **Delete** | Remove the page |
| **Set as Home** | Make this the default landing page |
| **View Settings** | Open detailed page configuration |

## Common Workflows

### Building a Multi-Page Application

1. Plan your page structure
2. Create all needed pages
3. Set the home page
4. Add content to each page
5. Configure navigation between pages

### Creating a Form Flow

1. Create a sequence of pages
2. Add form elements to each
3. Set up navigation between steps
4. Configure final submission page

### Setting Up a Dashboard

1. Create a dashboard page
2. Connect to data sources
3. Add tiles with visualizations
4. Configure data bindings

---

## Best Practices

1. **Plan page structure first** - Map out your pages before building

2. **Use descriptive names** - "User Profile" is clearer than "Page 1"

3. **Keep page count manageable** - Don't create pages unnecessarily

4. **Consider navigation** - How will users move between pages?

5. **Set meaningful URLs** - Use SEO-friendly URL slugs

6. **Test on all pages** - Verify each page works correctly

---

## Troubleshooting

### Page Not Saving

- Check for validation errors
- Ensure required fields are filled
- Verify network connection

### Page Not Displaying Content

- Confirm tiles are added to the page
- Check element visibility conditions
- Review page-level conditions

### Duplicate Page Names

- Rename one of the duplicates
- Use more specific naming
- Check for case sensitivity issues
